# 🕹️ 맞짱 오락실 (tenten-arcade)

텐텐 오락실 스타일의 파티 미니게임 앱.
**폰 하나로 마주 보고 하는 2인 대전 8종 + 방 코드로 모이는 온라인 다인전 14종 = 총 22게임.**

React Native (Expo)로 만들어서 iOS/Android 모두 Expo Go로 바로 실행할 수 있고,
온라인 대전은 저장소에 포함된 경량 WebSocket 서버(Node.js)로 돌아갑니다.

---

## 🚀 빠른 시작 (클론 후 처음 실행)

요구사항: **Node.js 20+**, 폰에 **Expo Go 앱** (App Store / Play스토어)

```bash
git clone <이 저장소 URL>
cd tenten-arcade

# 1) 앱 의존성 설치 + 실행
npm install
npx expo start
# → 터미널에 뜨는 QR코드를 폰의 Expo Go로 스캔

# 2) (온라인 대전 쓸 때만) 서버 실행 — 새 터미널에서
cd server
npm install
npm start        # 포트 8790
```

- 폰과 PC가 **같은 와이파이**에 있으면 끝. 앱의 "🌐 온라인 대전"에 서버 주소가 자동 입력됩니다 (Expo의 Metro 호스트 IP를 그대로 사용).
- Windows에서는 `서버켜기.bat` 더블클릭으로도 서버를 켤 수 있습니다.
- 브라우저로 보고 싶으면 `npx expo start --web`.

## 🌐 온라인 대전 사용법

1. PC에서 서버 실행 (`cd server && npm start`)
2. 각자 폰에서 Expo Go로 앱 실행 → **🌐 온라인 대전**
3. 한 명이 **방 만들기** → 화면의 4자리 코드를 친구들이 입력해 참가
4. 방장(👑)이 게임 선택 → 자동 시작. 게임이 끝나면 로비로 돌아와 다른 게임 선택

### 같은 와이파이가 아닐 때 (진짜 인터넷 플레이)

`server/`는 의존성이 `ws` 하나뿐인 단일 파일 Node 서버라 아무 데나 배포하면 됩니다.

1. [Render](https://render.com), Railway, Fly.io 등에 `server/` 폴더를 배포 (start command: `node index.js`, `PORT` 환경변수 자동 인식)
2. 앱 로비 하단 "서버 주소"에 배포 주소 입력: `wss://내앱이름.onrender.com`

## 📁 프로젝트 구조

```
tenten-arcade/
├── App.js                 # 홈 화면 + 게임 라우팅 (로컬 게임 목록: GAMES 배열)
├── src/
│   ├── theme.js           # 색상 팔레트
│   ├── components/ui.js   # 공용 Screen/BigButton/Chip
│   ├── games/             # 로컬 2인 대전 게임 (파일 1개 = 게임 1개)
│   └── online/
│       ├── net.js         # WebSocket 래퍼, 서버 주소 자동감지, 시드 난수
│       ├── OnlineLobby.js # 방 만들기/참가/게임 선택 (온라인 목록: ONLINE_GAMES 배열)
│       └── *.js           # 온라인 게임들 (파일 1개 = 게임 1개)
├── server/
│   └── index.js           # 방 코드 기반 릴레이 서버 (dots/splat만 서버 판정)
└── 서버켜기.bat            # Windows용 서버 실행 바로가기
```

**게임 추가/삭제**: 로컬은 `App.js`의 `GAMES` 배열, 온라인은 `OnlineLobby.js`의 `ONLINE_GAMES` 배열에서 한 줄 추가/삭제하면 끝.

### 온라인 게임 구조 (새 게임 만들 때)

- 서버는 기본적으로 **릴레이**만 합니다: 클라이언트가 `{t:'msg', d:{...}}`를 보내면 같은 방 사람들에게 `{t:'msg', from, d}`로 전달
- 시작 시 서버가 주는 `seed`로 `mulberry32(seed)` 난수를 쓰면 **전원 동일한 판**(노트 차트, 과녁 위치, 운석 궤적 등)이 만들어짐
- 동시 획득 충돌이 나는 게임(점 먹기, 페인트)만 서버가 판정 (`server/index.js`의 `eat`/`paint` 케이스 참고)
- 각자 화면형 게임은 공통 패턴: `count → go → waiting → results` + 진행도 릴레이

## 🛠 기술 스택

- **앱**: Expo SDK 57 / React Native 0.86 / React 19 — 외부 게임엔진 없이 RN 애니메이션·타이머로 구현
- **서버**: Node.js + `ws` (단일 파일, ~150줄)
- 그래픽 에셋 없음 (이모지 기반) — 클론 후 바로 실행 가능
